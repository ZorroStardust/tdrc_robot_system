### 1. 测试理想编号与实际编号以及理想方向与实际方向

使用脚本：interactive_pmac_joint_test.py

在PMAC IDE激活主站后，运行脚本，首次打开必须勾选run hardware_boot before Modbus connect，否侧总是连接失败。

连接成功后，我使用Manual Motor Calibration来验证电机编号和方向。
Manual Motor Calibration 左侧显示的编号我无法确定是理性编号还是实际编号，但是通过观察电机转动方向和编号的关系可以进行推断。

首先，在运行脚本前，将编号映射和方向映射均改为默认值，即 MOTOR_INDEX_MAP = {1: 1, 2: 2, 3: 3, 4: 4} 和 MOTOR_DIRECTION_MAP = {1: 1, 2: 1, 3: 1, 4: 1}。观察时，以竖直向上为z轴正方向，水平向右为x轴正方向，水平向里为y轴正方向。

注意，该环节仅测试电机，并未加上绕线机构，避免测试时飞车。

#### 测试记录

1. motor1
   - 发送角度： + 180 deg
   - 观察：实际编号1的电机绕理想z轴反转180deg
2. motor2
   - 发送角度： + 180 deg
   - 观察：实际编号2的电机绕理想z轴正转180deg
3. motor3
    - 发送角度： + 180 deg
    - 观察：实际编号3的电机绕理想z轴反转180deg
4. motor4
    - 发送角度： + 180 deg
    - 观察：实际编号4的电机绕理想z轴反转180deg

因此，映射设置应当为：

```python
MOTOR_INDEX_MAP = {1: 2, 2: 1, 3: 3, 4: 4}
MOTOR_DIRECTION_MAP = {1: -1, 2: 1, 3: -1, 4: -1}
```
设置好以上参数后，再次运行脚本，观察电机转动方向是否符合预期。
发现motor1-4显示的仍然是实际1-4，转动方向也没变，有两种可能：1）映射设置无效；2）映射设置有效，但是ui的电机校准界面显示的编号始终是实际电机编号，且电机方向也以实际电机转动方向为准（方向应该是底层pmac设置的）。

如果是后者，则最好再ui中的Manual Motor Calibration界面上标注一下是实际编号还是理想编号，避免后续测试人员的混淆。并且加入一个切换按钮，切换显示实际编号和理想编号及其对应的转动方向。

如果是前者，则需要检查代码中是否正确使用了 MOTOR_INDEX_MAP 和 MOTOR_DIRECTION_MAP 进行编号和方向的映射。

### 2. 电机运行状态

#### 异常声响
电机目前的速度和加速度参数都一致，motor1的状态相比其他电机存在明显差异，电机转动过程中有沙沙作响的声音，电机到达目标位置后，内部仍然发出沙沙声，可能是减速机？而且发送一个实际角度负数时，到达目标位置时会发出高频的尖锐声音，如果不是机械问题，怀疑电机三环参数可能存在问题。

其余电机运行状态尚可，转动平稳，无异常声响。

#### 初始状态存在不安全隐患

目前存在一个最大的不安全隐患，即电机初始状态不受控制，可能存在飞车风险。
具体而言，假设当前测试时，我通过这个交互式的代码给电机发送了一个目标位置，但是由于某些原因，电机没有到达这个目标位置，然而目标位置已经写入了某个寄存器，可能是PMAC，也可能是电机驱动器（有待确认），总之在这样的情况下，当我尝试通过重启来解决电机不转的问题时，发送hardware_boot命令会将之前写入的目标位置重新发送给电机，导致电机飞车，存在安全隐患。
因此，建议在hardware_boot命令中加入一个清除目标位置的步骤，可以参考电机CSP模式的标准流程，将当前位置作为目标位置写入寄存器。
